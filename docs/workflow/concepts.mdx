El flujo sigue el **atom-ai-toolkit**: skills encadenados con outputs acumulativos en `.atom-ai/outputs/`. La fuente de verdad son los archivos en `packages/toolkit/skills/` y `skills/_registry.yml`.

## Mapa interactivo

Abre **[http://localhost:4321/workflow](http://localhost:4321/workflow)** (o tu URL de producción + `/workflow`).

- Clic en un nodo **skill** → pestaña **Skill** (contenido de `SKILL.md`).
- Pestaña **Ejecutar** → input + provider/model → **Run skill** (requiere API con claves LLM en `.env`).
- Pestaña **Output** → markdown generado del run.

## Sincronizar mapa en docs/web

Tras cambiar `_registry.yml` o posiciones del generador:

```bash
npm run sync-workflow
```

Eso regenera `apps/api/data/default.json` y copia `workflow-map.json` a `docs/public` y `apps/web/public`.

## CLI (toolkit)

Desde la raíz del monorepo (con `@atom-ai/toolkit` instalado):

```bash
npx atom-ai init --project "mi-feature"   # si usas el bin del toolkit en packages/toolkit
```

La web usa el API (`POST /api/v1/projects`, `POST .../run/:skillId`) con proyectos bajo `apps/api/data/projects/`.
